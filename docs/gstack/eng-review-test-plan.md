## Affected Pages/Routes

- `/` — 选择官方精选话题、基于模板改写问题、选择默认 4 人格、切换热度奖励开关、启动实验
- `/experiments/[id]` — 查看双世界对照结果、10 轮时间线、3 阶段总结、拐点取证和反事实救援分支

## Key Interactions to Verify

- 在首页选择一个精选话题并轻度改写后，能成功启动一次实验
- 切换 `热度奖励开/关` 后，双世界结果能稳定呈现出可解释的差异
- 相同实验配置重复运行时，结果能复现或高度接近
- 结果页能正确展示左右世界的 10 轮内容、阶段总结和关键转折点
- 用户点击某个拐点后，能看到对应的取证说明
- 用户触发一次反事实救援后，能看到基于同一基线规格的救援世界结果
- 刷新 `/experiments/[id]` 后，已保存实验结果仍然一致可读

## Edge Cases

- 话题改写为空、过长、包含异常字符时，首页应阻止提交并显示清晰错误
- 用户快速双击“开始实验”时，不应创建两个实验记录
- 生成中途失败时，用户应看到明确失败状态和重试入口，而不是空白页
- 实验记录存在但某个世界数据缺失或 JSON 结构不匹配时，结果页应展示可恢复的错误态
- 同一实验配置在重新运行时，如果内容包版本不同，应明确提示版本变化导致结果不可直接对比
- 反事实救援只能基于已保存的基线实验触发，不能对不存在或损坏的实验记录静默失败

## Critical Paths

- 首页创建实验 → 实验编排器生成双世界 → 单事务持久化 → 跳转到结果页
- 结果页按实验 ID 读取不可变记录 → 展示双世界对照、阶段总结、拐点和救援入口
- 用户在同一实验上触发一次单点干预 → 生成救援世界 → 与原实验结果对照展示
- 相同配置重复运行 → 依据已保存的种子、规则版本和内容包版本复现结果
