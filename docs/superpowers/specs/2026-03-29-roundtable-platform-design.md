# Roundtable Platform Design

**目标：** 把当前“展示型实验台”改成“圆桌讨论 + 每轮平台动作 + 终局结案页”的交互体验，让用户亲手把普通讨论推向失控。

## 设计结论

- 首页改为圆桌主舞台，不再先展示完整报告。
- 用户扮演平台操盘手，每轮只做一次平台动作选择，不直接发言。
- 首版采用 `6` 轮推进，保证每轮反馈明确且不拖沓。
- 每轮固定返回三类反馈：圆桌新发言、机制指标变化、平台奖励说明。
- 结果页改成结案页，顺序固定为：终局判定 -> 操作回放 -> 表达分化 -> 证据回看。

## 首页结构

### 1. 圆桌主舞台

- 展示一个普通但有张力的话题。
- 显示 `4` 个固定角色围桌发言。
- 当前轮次只突出新增发言，历史轮次压缩成前情轨迹。

### 2. 平台动作面板

- 每轮提供 `4` 个动作：
  - 推高情绪
  - 放大对立
  - 奖励猎奇
  - 压低细节
- 用户点选动作后，当前轮锁定，系统推进下一轮。

### 3. 机制指标

- 常驻 `3` 个指标：
  - 情绪密度
  - 身份对立
  - 问题偏离度
- 指标每轮变化，作为“层层递进”的可视反馈。

### 4. 阶段轨迹

- 固定四段：
  - 正常讨论
  - 立场固化
  - 表演升级
  - 原问题消失
- 随轮次推进逐步点亮。

## 轮次节奏

### 第 1 轮

- 系统给出初始讨论，仍在围绕问题。
- 用户第一次选择平台动作。

### 第 2 轮

- 角色开始感知奖励逻辑。
- 指标首次显著变化。

### 第 3 轮

- 立场开始成形，姿态感增强。

### 第 4 轮

- 冲突升级，讨论更像对观众表演。

### 第 5 轮

- 原问题失焦，身份和情绪压过问题。

### 第 6 轮

- 终局定型，并生成结案页。

## 结果页结构

### 1. 终局判定

- 直接说明这场讨论最后变成了什么。

### 2. 失控路径回放

- 回放用户每轮的平台动作与对应偏移结果。

### 3. 被奖励的表达 / 被挤掉的表达

- 对照展示平台最终放大的表达类型与先消失的表达类型。

### 4. 圆桌证据回看

- 提供完整 `6` 轮证据回放，作为结论的证据层。

## 首版实现范围

- 只实现精选样本话题，不引入知乎/微博采集。
- 只实现单线圆桌演化，不再保留双世界并行展示。
- 保留本地存储与结果页路由，确保实验结果可回看。
