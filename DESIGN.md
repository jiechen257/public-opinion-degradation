# Agent Hub Design System

## 产品定位

`Agent Hub` 不是论坛，也不是 AI 聊天工具。

首版是一个 `证据板 / 双世界实验台 / 互联网舆论解剖台`，用本地 mock 的方式把同一个问题在不同平台奖励逻辑下如何劣化显影出来。

## 设计目标

- 首页先让用户看懂“这是一场舆论显影实验”，再暴露控制项。
- 结果页先给结论，再给证据，不把页面做成聊天记录墙。
- 双世界差异必须依靠文案标签、层级和因果说明成立，不能只靠颜色。
- 页面即使去掉一部分装饰，仍然要靠排版、字体和结构成立。

## 视觉母语

### 关键词

- 纸面证据
- 调查结论板
- 讨论平台遗骸
- 冷热对照

### 字体

- 正文使用稳定、可读的中文无衬线字体。
- 标题使用更有叙事感的中文衬线字体，承担“结论”和“引语”的重量。

### 颜色

- 纸面底色：暖米色、旧纸色，不使用纯白 SaaS 背景。
- 墨色正文：深棕黑，避免纯黑的数字看板气质。
- 红色系：用于失控、偏移、热度放大。
- 冷蓝系：用于对照、迟缓世界、反事实救援。
- 颜色只做辅助，不承担唯一的信息区分职责。

### 背景与层次

- 使用细网格、轻微径向晕染和纸面质感建立氛围。
- 主要层级依赖边框、留白、版式和字体对比，不依赖统一阴影卡片。

## 首页规范

### 信息层级

1. 今日样本问题
2. 双世界分叉预告
3. 主 CTA
4. 次级控制项：话题改写、人格组合、热度奖励开关

### 版式原则

- 首屏必须先演示、后控制。
- 主舞台应围绕问题样本和分叉预告组织，不用通用 hero 口号起手。
- 次级控制区可以下沉到首屏以下，或在移动端折叠。

### 首页状态

- 创建中：叙事式等待，逐步展示“正在生成两个世界 / 正在寻找第一句带偏的话 / 正在推演救援分支”。
- 空态：明确告知暂无可观察样本，并提供回到默认示例的入口。
- 错误态：显示失败原因和重试入口，不给空白页。

## 结果页规范

### 固定顺序

1. 双世界差异摘要
2. 关键拐点
3. 反事实救援
4. 完整时间线证据

### 版式原则

- 顶部像调查结论板，不像仪表盘，也不像聊天页。
- 时间线是证据层，应下沉在结论之后。
- 双世界对照必须有明确标签、标题和平台奖励说明。

### 结果页状态

- 读取中：优先保留摘要区骨架，避免整页跳动。
- 缺失：实验存在但无有效结果时，展示可恢复空态。
- 错误：实验 ID 不存在、记录损坏或版本不兼容时，展示明确错误说明。
- 部分成功：若只有一个世界可读，顶部必须明确说明无法形成完整对照，并降级为单世界证据阅读。

## 响应式

- 桌面首页：主舞台 + 次级证据轨的非对称布局。
- 手机首页：单列叙事流，顺序保持为样本问题、分叉预告、阶段条、主 CTA、折叠控制区。
- 桌面结果页：允许同时展示双世界摘要和双列证据。
- 手机结果页：保持顶层对照摘要不变，详细证据改为单世界详情切换。

## 可访问性

- 所有主要点击目标不低于 `44px`。
- 键盘顺序遵循叙事顺序：问题样本 -> CTA -> 控制项 -> 摘要 -> 拐点 -> 救援 -> 时间线。
- 焦点态清晰可见，不能只依赖浏览器默认样式。
- 生成中的叙事文案要能被辅助技术读出。
- 时间线使用列表语义，至少可读出“世界 / 轮次 / 阶段 / 发言摘要”。

## Anti-Slop 黑名单

- 居中 hero + 下方一排功能卡
- 三列 feature grid
- 蓝紫渐变 AI 配色
- 统一阴影卡片堆层级
- 仪表盘式结果页
- 首页首屏把全部控制项一次性摊开
- 双世界差异只靠颜色区分

## 实现落地检查

- 陌生人看首屏，应在数秒内理解这是“舆论如何坏掉”的显影实验。
- 去掉部分装饰后，页面依然能成立。
- 首页不能被轻易替换成“欢迎来到某某 AI”而不显违和。
